# Widget Extension Fix for Live Activities

## Problem
The ExpoWidgetsTarget extension was missing a Swift `@main` entry point, causing the build to fail validation with:
```
Invalid Mach-O header. The _swift5_entry section is missing for the "Payload/SevaEats.app/Plugins/ExpoWidgetsTarget.appex"
```

## Solution
This project includes an automated fix that ensures the widget extension always has a proper Swift entry point.

### What Was Fixed

1. **Added Swift Entry Point** (`ios/ExpoWidgetsTarget/index.swift`)
   - Added `@main` attribute
   - Created `ExpoWidgetsBundle` struct
   - Calls `ExpoWidgetsProvider.allWidgets()` to register Live Activities at runtime

2. **Build Configuration** (`ios/Podfile`)
   - Ensured dSYM generation for Release builds
   - Set proper Swift optimization levels
   - Added Swift version fallback

3. **Automated Fix Script** (`scripts/fix-widget-entry.sh`)
   - Automatically fixes the entry point after `expo prebuild`
   - Integrated into build workflows

### How It Works

The widget extension is generated by `expo-widgets` during prebuild, but it creates an incomplete file.
Our fix script runs automatically after prebuild to add the required Swift entry point.

### Building the App

#### For EAS Build (Recommended)
```bash
eas build --platform ios --profile production
```

The fix is automatically applied via the `prebuildCommand` in `eas.json`.

#### For Local Xcode Archive
```bash
# Run prebuild (includes automatic fix)
npm run prebuild

# Then in Xcode:
# Product > Archive
```

#### Manual Prebuild
```bash
npx expo prebuild --clean
npm run fix-widget
```

- `app.json` - Added widget plugin configuration and custom config plugin

- `ios/Podfile` - Added dSYM generation and Swift settings
- `package.json` - Added fix-widget script

### Live Activities

This app uses Live Activities for Dynamic Island to show meal delivery status.
The widget extension is required for Live Activities to work, even though we're not using traditional Home Screen widgets.

### Troubleshooting

If you encounter the Mach-O header error:

1. **After running `expo prebuild`**, always run:
   ```bash
   npm run fix-widget
   ```

2. **Check the file** `ios/ExpoWidgetsTarget/index.swift` contains:
   ```swift
   @main
   struct ExpoWidgetsBundle: WidgetBundle {
     @WidgetBundleBuilder
     var body: some Widget {
       ExpoWidgetsProvider.allWidgets()
     }
   }
   ```

3. **Verify in Xcode** that the ExpoWidgetsTarget scheme builds successfully

4. **For EAS builds**, the fix runs automatically - no manual intervention needed

### Why This Happens

The expo-widgets plugin (v55.0.0-alpha.3) generates a widget extension but leaves the entry point file incomplete when no traditional widgets are configured. Live Activities don't require widget definitions in `app.json`, but the extension still needs a valid Swift `@main` entry point to compile.
